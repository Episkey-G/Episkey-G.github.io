科学上网

一、Macos

现在Macos平台主流的科学上网客户端为ClashX

1.概述

​ Clash 是一个基于Go语言开发的跨平台代理程序。
​ ClashX 是 Clash Core 在 macOS 上的图形化程序，方便用户配置和使用。

​ 初次配置之前，推荐完整阅读一遍教程，再进行操作。

2.下载ClashX

  • 下载ClashX的客户端
  • 打开下载完成的 dmg 映像文件，将 ClashX 添加到应用程序文件夹中。

3.添加Clash配置订阅

次元连接

  • 前往官网 用户中心 复制 Clash 配置链接。
  • 打开 ClashX，点击菜单栏中的 ClashX 图标 打开菜单，进入 配置 > 托管配置 > 管理。
  • 在 托管的配置文件 窗口，点击 “添加”
  • 在 Url 一栏中输入 Clash 配置订阅链接，Config Name 为配置文件名，可以随意填写或留空。
  • 添加成功后 ClashX 会自动从链接下载配置文件，等待 “更新时间” 刷新后，即可关闭窗口。

4.设置为系统代理

  • 打开 ClashX 菜单中的 “设置为系统代理”，ClashX 便开始接管系统流量。

  • Clash 支持通过策略组，对不同的网站使用不同的策略。合理使用分流可以提升您的使用体验。

    🍃 Proxies 为节点策略组，可在其中选择节点。
    🍂 Domestic 决策中国大陆网站，默认 DIRECT 直连。
    ☁️ Others 决策没有收录进规则的网站，默认使用 🍃 Proxies 代理访问。

    ClashX 的默认出站模式为 “规则”，不推荐选择 “全局” 与 “直连”。

学习JavaWeb遇到的一些坑

  • 记录初学JavaWeb时遇到的问题
  • 写下解决办法，以便日后参考

tomcat

  1. 使用jstl包时，如果/usr/local/Cellar/tomcat@9/9.0.50/libexec/conf路径下的catalina.properties中配置为

那么就无法扫描到jstl包，导致500报错，无法解析资源

此时只需要将*.jar改为/，就可以避免这个报错，但是这样会使tomcat启动过慢。

resources

  1. 当导入外部资源文件时，路径正确，语法正确，资源文件却无法生效

    解决办法：

    ​ 建立resources文件，将文件导入进去之后，右键选择 Mark Dircetory as Resources Root，将其作为资源文件就可以成功解决无法引用外部资源的问题
